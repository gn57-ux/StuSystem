# analytics-dashboard — 需求规格

## 概述

实现数据看板、班级成绩概览、科目分析和重点关注学生列表，为教师提供考试后的班级整体视图。

## 项目信息

- 项目名: student-performance
- 架构类型: Turborepo monorepo（React + Hono/tRPC + Drizzle + PostgreSQL + Better Auth）

## 需求版本

| 日期 | 版本 | 说明 |
| --- | --- | --- |
| 2026-06-29 | v1 | 初始需求 |

## 用户故事

- 作为教师，我想要看到当前班级考试的关键指标摘要，以便快速了解整体情况
- 作为教师，我想要看到分数段分布和与上次考试的对比，以便识别班级变化趋势
- 作为教师，我想要看到各科目均分对比，以便发现薄弱科目
- 作为教师，我想要系统自动识别重点关注学生，以便及时干预

## 功能需求

1. [F-001] 统计计算：平均分、中位数、最高分、最低分、标准差、优秀率（≥满分×0.85）、及格率（≥满分×0.6）、低分率（<满分×0.4）
2. [F-002] 数据看板页：顶部数据卡片（学生数、平均总分、最高总分、及格率）+ 总分分布柱状图 + 科目均分对比柱状图 + 重点关注学生列表
3. [F-003] 班级分析页：分数段分布、优秀率/及格率环形图、与上次考试平均分对比条形图、进步 Top5、退步 Top5
4. [F-004] 科目分析页：所选科目的平均分/最高/最低、学生单科排名表、低分学生列表（单科低于班均15分）、波动较大学生
5. [F-005] 重点关注学生列表：按 5 类预警规则自动识别（见 PRD §10.6）
6. [F-006] 全局筛选联动：班级和考试切换时所有分析数据自动更新

## 非功能需求

- 性能：统计计算在数据库层（SQL 聚合）完成，不在应用层循环计算；图表首次渲染 <1s
- 安全：数据看板所有统计 API 用 `protectedProcedure`，teacher 可访问自己班级

## 验收标准

- [ ] [AC-001] 数据看板页显示 4 个关键指标卡片，数值与数据库中成绩一致
- [ ] [AC-002] 总分分布柱状图按分数段（0-59/60-69/70-79/80-89/90+）展示学生人数
- [ ] [AC-003] 科目均分对比柱状图展示所选考试全部科目的均分
- [ ] [AC-004] 班级分析页展示与上次考试（同班级最近一次已发布考试）的平均分对比
- [ ] [AC-005] 重点关注学生列表：至少命中「总分低位」和「单科薄弱」两类预警
- [ ] [AC-006] 切换班级/考试后所有图表和列表数据自动刷新

## 依赖

- Recharts（需添加到 apps/web 依赖）

## 开放问题

- 优秀率阈值是否固定（≥85%满分）还是可配置？→ MVP 固定，后续可配置
- 进步/退步 Top5 的依据是「排名变化」还是「总分变化」？→ 总分变化（绝对值）

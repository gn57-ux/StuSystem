# data-management — 需求规格

## 概述

实现班级、科目、学生三类基础数据的管理功能，包含 CRUD 操作和学生 CSV 批量导入。

## 项目信息

- 项目名: student-performance
- 架构类型: Turborepo monorepo（React + Hono/tRPC + Drizzle + PostgreSQL + Better Auth）

## 需求版本

| 日期 | 版本 | 说明 |
| --- | --- | --- |
| 2026-06-29 | v1 | 初始需求 |

## 用户故事

- 作为管理员，我想要创建和管理班级，以便组织学生
- 作为管理员，我想要配置科目，以便考试可以关联对应科目
- 作为管理员，我想要录入或批量导入学生名单，以便快速建立学生档案
- 作为教师，我想要搜索和筛选学生，以便找到特定学生的信息

## 功能需求

1. [F-001] 班级管理：列表展示（名称、年级、学生数）、新增、编辑、删除（无学生时才允许删除）
2. [F-002] 科目管理：列表展示、新增、编辑、删除（无关联考试时才允许删除）
3. [F-003] 学生列表：支持按班级筛选、按姓名/学号搜索、分页展示
4. [F-004] 学生详情：查看学生基础信息（姓名、学号、班级、性别、入学年份、状态、联系方式）
5. [F-005] 学生新增/编辑：表单录入，学号在同班级内不可重复
6. [F-006] 学生状态变更：在读 / 休学 / 转出
7. [F-007] 学生 CSV 批量导入：上传文件 → 解析预览 → 校验（学号重复、必填字段缺失）→ 确认导入

## 非功能需求

- 性能：学生列表支持分页（每页 20 条），避免全量加载
- 安全：班级/学生/科目的写操作需 `adminProcedure`（仅 admin 可修改基础数据）
- 兼容性：表格和模态框在 1280px 以上正常显示

## 验收标准

- [ ] [AC-001] 管理员可以创建、编辑、删除班级
- [ ] [AC-002] 管理员可以创建、编辑、删除科目
- [ ] [AC-003] 管理员可以新增学生，学号同班级内重复时展示错误提示
- [ ] [AC-004] 学生列表支持按班级筛选和关键词搜索，结果实时更新
- [ ] [AC-005] CSV 导入：上传后展示解析预览，有错误行时标红，确认后成功导入记录数
- [ ] [AC-006] 删除有学生的班级时展示阻止提示，不执行删除

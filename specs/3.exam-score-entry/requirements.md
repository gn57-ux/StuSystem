# exam-score-entry — 需求规格

## 概述

实现考试的创建与管理，以及在可编辑表格中录入、校验和提交成绩，支持 CSV 批量导入成绩。

## 项目信息

- 项目名: student-performance
- 架构类型: Turborepo monorepo（React + Hono/tRPC + Drizzle + PostgreSQL + Better Auth）

## 需求版本

| 日期 | 版本 | 说明 |
| --- | --- | --- |
| 2026-06-29 | v1 | 初始需求 |

## 用户故事

- 作为教师，我想要创建考试并关联参与班级和科目，以便后续录入成绩
- 作为教师，我想要在表格中直接编辑每个学生的每科成绩，并能用 Tab/Enter 快速切换，以便高效录入
- 作为教师，我想要看到超出满分或格式错误的成绩实时提示，以便避免录入错误
- 作为教师，我想要保存草稿再提交，以便分多次录入成绩
- 作为教师，我想要通过 CSV 文件批量导入成绩，以便从 Excel 迁移数据

## 功能需求

1. [F-001] 考试列表：展示考试名称、类型、日期、参与班级、状态；支持按状态筛选
2. [F-002] 创建考试：填写名称、类型、日期，选择参与班级（多选），选择参与科目（多选），配置每科满分
3. [F-003] 编辑考试：支持修改名称、日期，不允许修改已有成绩的考试的班级/科目配置
4. [F-004] 考试状态流转：草稿 → 录入中 → 已发布
5. [F-005] 成绩录入表格：行 = 学生，列 = 科目，单元格可直接编辑分数
6. [F-006] 成绩状态切换：每格支持 正常/缺考/免考/未录入，通过下拉或快捷键切换
7. [F-007] 实时校验：分数超出满分时单元格标红并提示；格式非数字时阻止输入
8. [F-008] Tab/Enter 切换：Tab 横向移动到下一科，Enter 纵向移动到下一学生
9. [F-009] 离开页面提醒：未保存草稿时离开页面弹出确认
10. [F-010] 草稿保存：调用 API 保存当前录入状态，不改变考试状态
11. [F-011] 提交发布：全部成绩录入完成后提交，考试状态变为「已发布」，系统触发排名计算
12. [F-012] CSV 成绩导入：上传 CSV（学号、科目名、分数、状态）→ 解析 → 预览校验 → 确认导入

## 非功能需求

- 性能：成绩录入表格按班级×科目加载，单次请求数据量 ≤ 200 行（按班级分页加载）
- 交互：单元格编辑响应 < 16ms（不触发额外网络请求），保存草稿去抖 2s

## 验收标准

- [ ] [AC-001] 教师可以创建含多个班级和科目的考试，每科满分可独立配置
- [ ] [AC-002] 成绩录入表格加载当前考试+班级的所有学生行
- [ ] [AC-003] 单元格输入后 Tab/Enter 正确移动焦点
- [ ] [AC-004] 超过满分的分数单元格标红并显示 tooltip 提示
- [ ] [AC-005] 保存草稿后刷新页面，数据保留
- [ ] [AC-006] 提交发布后考试状态变为「已发布」，无法再修改成绩
- [ ] [AC-007] CSV 导入：展示预览，错误行标红，成功导入后表格数据更新

# student-analysis — 需求规格

## 概述

实现学生个人学情详情页，展示长期成绩趋势、偏科分析、预警识别，并支持教师添加备注和处理预警。

## 项目信息

- 项目名: student-performance
- 架构类型: Turborepo monorepo（React + Hono/tRPC + Drizzle + PostgreSQL + Better Auth）

## 需求版本

| 日期 | 版本 | 说明 |
| --- | --- | --- |
| 2026-06-29 | v1 | 初始需求 |

## 用户故事

- 作为教师，我想要查看某个学生的历次考试成绩趋势，以便了解长期学情
- 作为教师，我想要看到学生的科目雷达图，以便直观识别优势和薄弱科目
- 作为班主任，我想要为重点关注学生添加备注和辅导建议，以便记录干预措施
- 作为教师，我想要确认或忽略系统预警，以便管理关注名单

## 功能需求

1. [F-001] 学生基础信息卡：姓名、学号、班级、性别、入学年份、状态
2. [F-002] 排名摘要：最近考试的总分、班级排名、总分与上次差值
3. [F-003] 各科最近成绩列表：科目名、得分、班均、排名
4. [F-004] 总分趋势折线图：历次考试（时间轴）的总分变化
5. [F-005] 单科趋势折线图：可切换科目，展示该科历次得分
6. [F-006] 科目雷达图：最近一次考试各科标准化得分（0-100 映射）
7. [F-007] 排名变化曲线：历次班级排名（Y 轴倒序，排名越低越靠上）
8. [F-008] 偏科与预警提示区：展示当前活跃预警（alertType + severity），可点击查看详情
9. [F-009] 教师备注列表：按时间倒序展示，含作者姓名
10. [F-010] 新增备注：文本输入框 + 提交，备注立即出现在列表

## 非功能需求

- 性能：趋势数据按学生 ID 查询，历次考试最多 20 次（足够 MVP）
- 安全：教师备注存储作者 ID，展示时 join user 表获取姓名；不能删除他人备注（MVP 暂不做删除）

## 验收标准

- [ ] [AC-001] 学生详情页展示基础信息卡和排名摘要
- [ ] [AC-002] 总分趋势折线图展示至少 2 次考试的数据点（含考试名称 tooltip）
- [ ] [AC-003] 科目雷达图展示最近一次考试的多科标准化分
- [ ] [AC-004] 预警区域展示来自 student_alerts 的活跃预警，点击可确认/忽略
- [ ] [AC-005] 教师可添加备注，提交后立即显示在列表顶部
- [ ] [AC-006] 确认或忽略预警后，该预警从活跃列表移除

## 依赖

- 4.analytics-dashboard（student_alerts 表有数据；analyticsRouter 复用）
- 1.foundation-schema（teacherNotes、studentAlerts schema）

## 开放问题

- 是否展示年级排名？→ MVP 暂不做，仅展示班级排名

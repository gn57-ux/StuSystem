# 成绩与学情分析系统 PRD

## 1. 产品概述

### 1.1 产品名称

成绩与学情分析系统

### 1.2 产品定位

一款面向学校教师、班主任和教务管理人员的 Web 管理系统，用于录入、管理和分析学生成绩，帮助教师快速识别学生学习趋势、薄弱科目、班级整体表现和需要重点关注的学生。

### 1.3 产品愿景

让成绩数据不只是表格和排名，而是变成可理解、可追踪、可行动的教学参考。

### 1.4 核心价值

- 降低成绩管理成本：统一管理学生、班级、考试、科目和成绩。
- 提升分析效率：自动生成班级概览、学生趋势、科目对比和排名变化。
- 支持教学决策：帮助教师识别薄弱知识方向和重点辅导对象。
- 形成学生学情档案：沉淀每名学生的阶段性成绩与学习表现。

## 2. 背景与问题

### 2.1 业务背景

学校和培训机构通常使用 Excel 或分散系统记录成绩。虽然 Excel 灵活，但在长期追踪、跨考试对比、班级分析、学生个体变化方面效率较低。教师需要一个轻量但专业的工具，把成绩数据组织成清晰的分析视图。

### 2.2 用户痛点

- 成绩录入和整理依赖 Excel，容易出错。
- 单次考试排名容易看，但长期趋势难追踪。
- 学生退步、偏科、波动异常难以及时发现。
- 班级平均分、优秀率、及格率等指标需要手动计算。
- 家长沟通和教学复盘缺少直观报告。
- 不同班级、不同考试之间对比成本高。

## 3. 目标用户

### 3.1 核心用户

**任课教师**

- 管理自己科目的成绩。
- 查看班级在该科目上的整体表现。
- 识别需要辅导的学生。

**班主任**

- 查看班级所有科目的综合表现。
- 关注学生总分、排名、偏科和退步情况。
- 用于家校沟通和班级管理。

**教务管理员**

- 管理班级、学生、教师、科目和考试。
- 查看年级或多个班级的整体数据。

### 3.2 次要用户

**学生或家长**

- 查看个人成绩趋势和学情报告。
- 了解优势科目、薄弱科目和改进建议。

MVP 阶段可暂不开放学生/家长端，仅保留后续扩展入口。

## 4. 产品目标

### 4.1 MVP 目标

在首版中完成以下闭环：

1. 管理班级、学生、科目和考试。
2. 录入或导入学生成绩。
3. 自动生成班级成绩概览。
4. 查看学生个人学情分析。
5. 查看科目分析和排名变化。
6. 标记需要关注的学生。

### 4.2 成功指标

- 成绩录入完成率：教师创建考试后成功录入成绩的比例 >= 85%。
- 分析页访问率：成绩录入后访问分析页的比例 >= 70%。
- 重点关注学生标记率：每次考试后至少 20% 班级产生关注学生名单。
- 数据导出使用率：每月有导出行为的活跃教师比例 >= 30%。

## 6. 用户角色与权限

| 角色 | 权限说明 |
| --- | --- |
| 超级管理员 | 管理学校基础设置、所有账号和全部数据 |
| 教务管理员 | 管理班级、学生、科目、考试和教师账号 |
| 班主任 | 查看和管理自己班级的学生、成绩和分析报告 |
| 任课教师 | 管理自己所授科目的成绩，查看相关班级分析 |
| 只读访客 | 仅查看被分享的报告或演示数据 |

MVP 可先实现管理员和教师两类角色。

## 7. 核心用户流程

### 7.1 初次配置流程

1. 管理员创建班级。
2. 管理员导入或录入学生名单。
3. 管理员配置科目。
4. 管理员创建教师账号。
5. 系统进入可录入成绩状态。

### 7.2 成绩录入流程

1. 教师进入考试管理。
2. 创建考试，例如“高一上学期期中考试”。
3. 选择考试涉及的班级和科目。
4. 录入或导入成绩。
5. 系统校验缺失值和异常分数。
6. 教师确认提交。
7. 系统生成分析结果。

### 7.3 学情分析流程

1. 教师进入数据看板。
2. 选择班级、考试、科目。
3. 查看班级整体表现。
4. 查看学生排名变化和趋势。
5. 打开重点关注学生详情。
6. 添加教师备注或辅导建议。
7. 导出报告。

## 10. 核心需求说明

### 10.1 学生管理

| 字段 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| 学生 ID | 系统生成 | 是 | 唯一标识 |
| 学号 | 文本 | 是 | 班级内不可重复 |
| 姓名 | 文本 | 是 | 学生姓名 |
| 性别 | 单选 | 否 | 男 / 女 / 未填写 |
| 班级 | 选择 | 是 | 所属班级 |
| 入学年份 | 数字 | 否 | 用于年级统计 |
| 状态 | 单选 | 是 | 在读 / 休学 / 转出 |
| 联系方式 | 文本 | 否 | 可选 |

### 10.2 考试管理

| 字段 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| 考试名称 | 文本 | 是 | 例如“期中考试” |
| 考试类型 | 单选 | 是 | 周测 / 月考 / 期中 / 期末 / 模拟考 |
| 考试日期 | 日期 | 是 | 用于趋势分析 |
| 参与班级 | 多选 | 是 | 至少一个班级 |
| 参与科目 | 多选 | 是 | 至少一个科目 |
| 满分规则 | 配置 | 是 | 每科满分，可不同 |
| 状态 | 单选 | 是 | 草稿 / 录入中 / 已发布 |

### 10.3 成绩录入

需求：

- 支持表格逐项录入。
- 支持 CSV / Excel 导入。
- 支持保存草稿。
- 支持提交前校验。
- 支持缺考、免考、未录入状态。
- 支持满分校验，例如数学满分 150，录入 160 时提示异常。

成绩状态：

| 状态 | 说明 |
| --- | --- |
| 正常 | 有有效分数 |
| 缺考 | 学生未参加考试，不计入部分统计 |
| 免考 | 特殊情况免考 |
| 未录入 | 当前成绩为空 |

### 10.4 班级分析

核心指标：

- 平均分
- 中位数
- 最高分
- 最低分
- 标准差
- 优秀率
- 及格率
- 低分率
- 分数段分布
- 班级排名
- 与上次考试相比变化

### 10.5 学生分析

核心指标：

- 总分趋势
- 单科趋势
- 班级排名变化
- 年级排名变化，可后续支持
- 优势科目
- 薄弱科目
- 偏科程度
- 进步幅度
- 退步幅度
- 成绩波动程度

### 10.6 预警规则

MVP 预警规则：

| 预警类型 | 判断规则 |
| --- | --- |
| 明显退步 | 本次班级排名较上次下降 10 名以上 |
| 单科薄弱 | 某科低于班级平均分 15 分以上 |
| 总分低位 | 总分位于班级后 20% |
| 波动较大 | 最近 3 次考试总分标准差超过阈值 |
| 偏科明显 | 最高科目标准化分与最低科目标准化分差距过大 |

预警结果应可被教师手动确认、忽略或备注。

## 13. 交互要求

### 13.1 管理后台体验

- 左侧导航固定。
- 顶部保留班级、考试、学期等全局筛选。
- 表格支持搜索、筛选、排序和分页。
- 关键数据卡片应能快速扫读。
- 图表需要有 hover tooltip。
- 页面不使用营销式大 hero，第一屏直接展示业务功能。

### 13.2 成绩录入体验

- 表格单元格可直接编辑。
- 支持键盘 Tab / Enter 快速切换。
- 异常分数实时提示。
- 未保存离开页面时提醒。
- 批量导入后展示校验结果。

### 13.3 报告体验

- 报告适合打印和 PDF 导出。
- 内容应简洁，优先展示关键结论。
- 教师评语支持手动编辑。

## 14. 视觉方向

### 14.1 产品气质

- 专业
- 清晰
- 稳定
- 数据可信
- 教育场景友好

### 14.2 UI 建议

- 使用管理后台布局：侧边栏 + 顶部筛选 + 内容区。
- 主色建议使用深蓝、青绿或中性蓝灰，搭配少量橙色用于预警。
- 数据卡片不要过大，保持信息密度。
- 表格和图表是核心，不要用过多装饰。
- 预警状态使用明确颜色，但避免全页面高饱和。

## 16. 隐私与安全

学生成绩属于敏感教育数据，需要注意：

- 登录后才能访问系统。
- 不同角色只能访问授权班级和学生。
- 导出报告需要记录操作日志。
- 删除学生或考试前需要二次确认。
- 不在公开链接暴露完整学生信息。
- 后续可加入数据脱敏和审计日志。

## 17. 边界情况

| 场景 | 处理方式 |
| --- | --- |
| 学生缺考 | 标记为缺考，不计入平均分，可单独统计 |
| 成绩超过满分 | 实时提示，禁止提交或要求确认 |
| 导入文件缺少学生 | 展示未匹配名单，允许跳过或手动匹配 |
| 学生重名 | 使用学号作为主匹配字段 |
| 考试未录完 | 状态保持为录入中，不生成正式报告 |
| 没有历史考试 | 趋势图展示空状态，不显示退步预警 |
| 班级人数过少 | 部分统计指标提示样本不足 |

## 19. 验收标准

MVP 完成时应满足：

- 管理员可以创建班级、学生、科目和考试。
- 教师可以为一次考试录入多个科目的成绩。
- 系统可以计算平均分、最高分、最低分、及格率和优秀率。
- 系统可以展示学生个人成绩趋势。
- 系统可以展示班级分数段分布和科目均分对比。
- 系统可以识别至少 3 类重点关注学生。
- 用户可以导出成绩数据。
- 桌面端可正常使用，核心页面在平板宽度下不出现严重布局问题。
